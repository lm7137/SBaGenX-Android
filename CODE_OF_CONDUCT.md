# Code of Conduct

## Our standard

This project expects respectful, technically serious collaboration.

Examples of behavior that contribute to a productive environment include:

- discussing ideas and bugs in good faith
- giving clear, specific technical feedback
- accepting correction when evidence shows a mistake
- focusing criticism on code, behavior, documentation, and decisions rather than on people
- helping new contributors find useful entry points

Examples of unacceptable behavior include:

- personal attacks
- harassment or intimidation
- deliberately inflammatory or bad-faith engagement
- publishing private information without permission
- repeated refusal to engage with evidence while disrupting the project

## Scope

This applies in project spaces, including issues, discussions, pull requests, and related community channels.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, discussions, or contributions that violate this code of conduct.
